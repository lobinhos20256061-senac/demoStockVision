# 🧪 Lista de Testes para Alunos - StockVision 2.0

Esta checklist foi pensada para ser executada pelos alunos após realizarem o login na aplicação. O objetivo é validar os principais fluxos do sistema e registrar se cada funcionalidade está funcionando corretamente.

## Instruções

- Acesse a aplicação e faça login com as credenciais fornecidas.
- Para cada teste, siga os passos descritos.
- Marque como "OK" quando o comportamento esperado ocorrer.
- Se algo não funcionar, registre o erro observado.

---

## 1. Acesso e autenticação

- [ ] Fazer login com usuário e senha válidos.
- [ ] Verificar se o sistema redireciona corretamente para o dashboard.
- [ ] Verificar se a sessão permanece ativa durante a navegação.
- [ ] Fazer logout e confirmar que a sessão é encerrada.

Resultado esperado: o aluno consegue entrar na aplicação e navegar entre as telas sem erros.

---

## 2. Dashboard

- [ ] Acessar a tela inicial após o login.
- [ ] Verificar se os cards de resumo aparecem na página.
- [ ] Confirmar se o gráfico ou visualização principal carrega corretamente.
- [ ] Verificar se as informações exibidas fazem sentido com o conteúdo cadastrado no sistema.

Resultado esperado: o dashboard carrega com dados visuais e métricas coerentes.

---

## 3. Cadastro e visualização de produtos no estoque

- [ ] Acessar a tela de Estoque.
- [ ] Criar um novo produto preenchendo nome, SKU, categoria e preço.
- [ ] Salvar o cadastro e verificar se o produto aparece na lista.
- [ ] Clicar no produto para visualizar os detalhes.

Resultado esperado: o produto é cadastrado e aparece na tabela de estoque com os dados corretos.

---

## 4. Filtros da tela de estoque

- [ ] Na tela de Estoque, utilizar o campo de busca por nome ou SKU.
- [ ] Aplicar filtro por categoria.
- [ ] Aplicar filtro por status.
- [ ] Informar um valor máximo de preço e verificar o resultado.
- [ ] Limpar os filtros e confirmar que a lista volta ao estado original.

Resultado esperado: a tabela é filtrada corretamente de acordo com os critérios escolhidos.

---

## 5. Edição e exclusão de produtos

- [ ] Selecionar um produto já cadastrado.
- [ ] Editar pelo menos um campo e salvar as alterações.
- [ ] Confirmar se a alteração apareceu na lista.
- [ ] Excluir um produto e verificar se ele some da tabela.

Resultado esperado: a atualização e a remoção ocorrem corretamente.

---

## 6. Logística reversa

- [ ] Acessar a tela de Logística Reversa.
- [ ] Verificar se o campo de produto apresenta opções carregadas.
- [ ] Preencher os campos do formulário.
- [ ] Enviar o formulário e verificar se a operação foi concluída.
- [ ] Confirmar se o item ficou registrado na listagem de reversa.

Resultado esperado: o formulário aceita os dados e o item aparece no histórico/registro da logística reversa.

---

## 7. Previsão de demanda

- [ ] Acessar a tela ou módulo de demanda.
- [ ] Gerar ou visualizar a previsão referente a um produto.
- [ ] Verificar se o valor exibido está coerente com os dados apresentados.

Resultado esperado: a previsão é exibida sem erros e com valor legível.

---

## 8. Módulo ESG

- [ ] Acessar a tela de ESG.
- [ ] Verificar se as métricas ou indicadores aparecem corretamente.
- [ ] Confirmar se os dados são apresentados de forma clara e organizada.

Resultado esperado: as informações ESG carregam corretamente e são compreensíveis.

---

## 9. Parceiros e fornecedores

- [ ] Acessar a tela de Parceiros.
- [ ] Verificar se a lista de parceiros é exibida.
- [ ] Criar ou visualizar um parceiro, se a funcionalidade estiver disponível.

Resultado esperado: o módulo lista e manipula os dados de parceiros sem falhas.

---

## 10. Responsividade e experiência visual

- [ ] Testar a navegação em tela grande.
- [ ] Testar a navegação em tela menor, como celular ou tablet.
- [ ] Verificar se os botões, menus e formulários continuam acessíveis.
- [ ] Confirmar se não há elementos quebrados ou sobrepostos.

Resultado esperado: a interface permanece funcional e organizada em diferentes tamanhos de tela.
